# Page types and adaptation notes

Use different standards for different page types.

## Open-source README
Optimize for:
- clarity
- credibility
- fast onboarding
- star-worthiness
- visible install path

## Developer landing page
Optimize for:
- clear value proposition
- problem/solution framing
- concrete proof
- obvious CTA

## Docs homepage
Optimize for:
- orientation
- navigation
- getting started path
- task discovery

## Launch page
Optimize for:
- strong angle
- compressed explanation
- proof and novelty
- easy sharing

## Feature page
Optimize for:
- one specific job to be done
- mechanism clarity
- proof of usefulness
- next action
