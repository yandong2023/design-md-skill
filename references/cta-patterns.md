# CTA patterns

Choose CTAs based on the page goal.

## Common primary CTAs

### For open-source repos
- Star on GitHub
- Try the demo
- Read the docs
- Get started

### For docs homepages
- Start here
- Install now
- Follow the quickstart
- Explore examples

### For product landing pages
- Start free
- Book a demo
- Try it locally
- See how it works

## CTA rules

- One primary CTA per hero by default
- Secondary CTA is optional, not mandatory
- Place the first CTA above the fold
- Use action verbs
- Match CTA wording to user readiness

## CTA mistakes

Avoid:
- too many equal-weight CTAs
- vague labels like “Learn more” when a stronger action is available
- promising a demo when none exists
- putting install too deep for developer audiences
