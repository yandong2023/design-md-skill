# Landing page review checklist

Use this short checklist before rewriting a page.

## First screen
- Can a new visitor explain what this product is in one sentence?
- Is the audience obvious?
- Is there a visible CTA?
- Is there any proof signal near the top?

## Structure
- Does the page explain the problem before listing features?
- Is there a clean flow from problem → solution → proof → action?
- Are the section titles specific?

## Trust
- Does the copy avoid inflated claims?
- Are benchmarks or claims actually supported?
- Does the page sound like a product, not internal notes?

## Developer fit
- Is install / quickstart easy to find?
- Is the language concrete?
- Are examples visible soon enough?
